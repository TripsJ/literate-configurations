* Autostart Script
:PROPERTIES:
:CUSTOM_ID: autostart
:END:

The autostart script lives at ~~/.config/qtile/scripts/autostart.sh~ and is
called exactly *once* by Qtile through the ~startup_once~ hook (see the Hooks
section).  It is responsible for launching background daemons, setting the
wallpaper, compositing, notification daemon, and any other session-level
services that must not be duplicated across restarts.

Keep this file idempotent where possible — i.e. guard daemon launches with
~pgrep~ checks so that a Qtile restart (~mod+shift+r~) does not spawn duplicate
processes. this is done using the run function in the environment block

** shebang and environment
This defines the run function to guard daemons and sets the keyboard layout
#+NAME: autostart-shebang
#+BEGIN_SRC shell
#!/usr/bin/env bash
# autostart.sh — launched once at Qtile startup
# Edit this file via its org source: qtile-config.org
function run {
  if ! pgrep -x $(basename $1 | head -c 15) 1>/dev/null;
  then
    $@&
  fi
}

* Colour Themes (~themes.py~)
:PROPERTIES:
:CUSTOM_ID: themes
:END:

All colour values are centralised in ~themes.py~.  The module exposes a single
public function, ~get_theme(name)~, which returns an ordered list of hex-colour
strings.  ~config.py~ imports from this module, so changing a theme requires
touching only one file.

Each palette is stored as a dict keyed by integer index.  The ~get_theme~
function sorts by key before returning, guaranteeing a stable positional
mapping even if you add colours in non-sequential order.

** get_theme function
The ~get_theme~ function is deliberately simple.  It raises a ~ValueError~ for
unknown theme names rather than silently falling back, so misconfiguration is
caught at startup rather than producing subtly wrong colours at runtime.
#+NAME: themes-get-theme
#+BEGIN_SRC python
def get_theme(theme_name):
    """Return an ordered list of [hex, hex] colour pairs for *theme_name*.

    Colours are sorted by their integer key so that positional indexing in
    config.py (e.g. colors[8]) is stable and predictable.

    Raises ValueError if the theme name is not found.
    """
    theme = THEMES.get(theme_name)
    if not theme:
        raise ValueError(f"Theme '{theme_name}' not found.")
    return [theme[i] for i in sorted(theme.keys())]
#+END_SRC

* Battery Widget (~arcobattery2.py~)
:PROPERTIES:
:CUSTOM_ID: arcobattery
:END:

~arcobattery2.py~ provides two custom Qtile widgets: ~Battery~ (text-based) and
~BatteryIcon~ (image-based).  Both inherit from ~_Battery~, a private base class
that handles reading the kernel's power supply interface under
~/sys/class/power_supply/~.

The file is kept as a self-contained module so it can be copied to any Qtile
install without modification.  Place it in ~~/.config/qtile/~ alongside
~config.py~ and import it with ~from arcobattery2 import BatteryIcon~.

** imports and constants
The battery name (~BAT_NAME~) defaults to ~BAT1~.  If your machine exposes a
different name (e.g. ~BAT0~, common on Thinkpads), override it here or set it
via the ~battery_name~ widget parameter at instantiation time.
#+NAME: arcobattery-imports
#+BEGIN_SRC python
from __future__ import division

import cairocffi
import os
from libqtile import bar
from libqtile.widget import base
from pathlib import Path

# Default battery name as seen under /sys/class/power_supply/.
# Check with: ls /sys/class/power_supply/
BAT_NAME = "BAT1"

BAT_DIR = '/sys/class/power_supply'

# Status strings returned by the kernel's 'status' file.
CHARGED     = 'Full'
CHARGING    = 'Charging'
DISCHARGING = 'Discharging'
UNKNOWN     = 'Unknown'

# Map of logical parameter names to the possible kernel file names that
# provide each value.  The list is tried in order; the first readable file wins.
BATTERY_INFO_FILES = {
    'energy_now_file':  ['energy_now', 'charge_now'],
    'energy_full_file': ['energy_full', 'charge_full'],
    'power_now_file':   ['power_now', 'current_now'],
    'status_file':      ['status'],
}
#+END_SRC

** Battery base class
~_Battery~ is not intended to be instantiated directly.  It provides the
file-reading machinery (~_load_file~, ~_get_param~, ~_get_info~) shared by both
concrete widgets.

The caching dict ~self.filenames~ avoids redundant filesystem probes after the
first successful read: once the correct file name is found for a given
parameter, it is stored and reused on every subsequent poll.
#+NAME: arcobattery-base-class
#+BEGIN_SRC python
class _Battery(base._TextBox):
    """Base battery class — do not instantiate directly."""

    filenames = {}

    defaults = [
        ('battery_name',     BAT_NAME, 'ACPI name of a battery, usually BAT0'),
        ('status_file',      'status', 'Name of status file in'
                                       ' /sys/class/power_supply/battery_name'),
        ('energy_now_file',  None,     'Name of file with the current energy'),
        ('energy_full_file', None,     'Name of file with the maximum energy'),
        ('power_now_file',   None,     'Name of file with the current power draw'),
        ('update_delay',     60,       'The delay in seconds between updates'),
    ]

    def __init__(self, **config):
        base._TextBox.__init__(self, "BAT", bar.CALCULATED, **config)
        self.add_defaults(_Battery.defaults)

    def _load_file(self, name):
        """Read a single file from the battery's sysfs directory."""
        try:
            path = os.path.join(BAT_DIR, self.battery_name, name)
            with open(path, 'r') as f:
                return f.read().strip()
        except IOError:
            if name == 'current_now':
                return 0
            return False
        except Exception:
            self.log.exception("Failed to get %s" % name)

    def _get_param(self, name):
        """Return the value for *name*, probing candidate file names if needed."""
        if name in self.filenames and self.filenames[name]:
            return self._load_file(self.filenames[name])
        elif name not in self.filenames:
            file_list = BATTERY_INFO_FILES.get(name, [])[:]
            if getattr(self, name, None):
                file_list.insert(0, getattr(self, name))
            for file in file_list:
                value = self._load_file(file)
                if value is not False and value is not None:
                    self.filenames[name] = file
                    return value
        self.filenames[name] = None
        return None

    def _get_info(self):
        """Return a dict with stat/now/full/power, or False on error."""
        try:
            info = {
                'stat':  self._get_param('status_file'),
                'now':   float(self._get_param('energy_now_file')),
                'full':  float(self._get_param('energy_full_file')),
                'power': float(self._get_param('power_now_file')),
            }
        except TypeError:
            return False
        return info
#+END_SRC

** Battery text widget
The ~Battery~ widget renders the battery state as formatted text.  The default
~format~ string shows a directional character (~^~ charging, ~V~ discharging), the
percentage, and the remaining time.  All of these can be overridden via keyword
arguments when instantiating the widget in ~config.py~.

The ~low_percentage~ threshold (default 10 %) triggers a colour change to
~low_foreground~ as a visual warning.
#+NAME: arcobattery-text-widget
#+BEGIN_SRC python
class Battery(_Battery):
    """A simple but flexible text-based battery widget."""

    orientations = base.ORIENTATION_HORIZONTAL
    defaults = [
        ('charge_char',     '^',                          'Character shown while charging'),
        ('discharge_char',  'V',                          'Character shown while discharging'),
        ('error_message',   'Error',                      'Message shown on read failure'),
        ('format',          '{char} {percent:2.0%} {hour:d}:{min:02d}', 'Display format'),
        ('hide_threshold',  None,                         'Hide text above this charge level'),
        ('low_percentage',  0.10,                         'Threshold for low-battery colour'),
        ('low_foreground',  'FF0000',                     'Colour used at low battery'),
    ]

    def __init__(self, **config):
        _Battery.__init__(self, **config)
        self.add_defaults(Battery.defaults)

    def timer_setup(self):
        update_delay = self.update()
        if update_delay is None and self.update_delay is not None:
            self.timeout_add(self.update_delay, self.timer_setup)
        elif update_delay:
            self.timeout_add(update_delay, self.timer_setup)

    def _configure(self, qtile, bar):
        if self.configured:
            self.update()
        _Battery._configure(self, qtile, bar)

    def _get_text(self):
        info = self._get_info()
        if info is False:
            return self.error_message
        try:
            if self.hide_threshold and \
                    info['now'] / info['full'] * 100.0 >= self.hide_threshold and \
                    info['stat'] != CHARGED:
                return ''
            elif info['stat'] == DISCHARGING:
                char = self.discharge_char
                time = info['now'] / info['power']
            elif info['stat'] == CHARGING:
                char = self.charge_char
                time = (info['full'] - info['now']) / info['power']
            else:
                return 'Full'
        except ZeroDivisionError:
            time = -1

        if time >= 0:
            hour = int(time)
            min  = int(time * 60) % 60
        else:
            hour = min = -1

        percent = info['now'] / info['full']
        if info['stat'] == DISCHARGING and percent < self.low_percentage:
            self.layout.colour = self.low_foreground
        else:
            self.layout.colour = self.foreground

        return self.format.format(char=char, percent=percent, hour=hour, min=min)

    def update(self):
        ntext = self._get_text()
        if ntext != self.text:
            self.text = ntext
            self.bar.draw()
#+END_SRC

** BatteryIcon widget
~BatteryIcon~ renders a PNG icon chosen from a set of named states (e.g.
~battery-70-charge~).  Icons are sourced from ~theme_path~ (defaults to
libqtile's bundled set) and can be overridden per-state via ~custom_icons~.

The ~setup_images~ method pre-scales all icons to fit the bar height at
configure time, so there is no per-frame scaling cost during drawing.

Note: there is a small logical bug in the original ~_get_icon_key~ method — the
~< .5~ branch for ~-60~ is unreachable because the ~< .6~ branch above it already
matches.  This is preserved as-is to maintain compatibility with existing
configs, but flagged here for awareness.
#+NAME: arcobattery-icon-widget
#+BEGIN_SRC python
class BatteryIcon(_Battery):
    """Battery life indicator widget — renders a theme-aware PNG icon."""

    orientations = base.ORIENTATION_HORIZONTAL
    defaults = [
        ('theme_path',   default_icon_path(), 'Path to the icon directory'),
        ('custom_icons', {},                  'Dict mapping state-key → filename overrides'),
        ('scaleadd',     0,                   'Additional scale factor (0 = disabled)'),
        ('y_poss',       0,                   'Vertical position offset in pixels'),
    ]

            self.length = 0

        self.current_icon = 'battery-missing'

            'battery-90',      'battery-90-charge',
            'battery-full',    'battery-full-charge',
            'battery-full-charged',
        )])
        self.icons.update(self.custom_icons)

    def timer_setup(self):
        self.update()
        self.timeout_add(self.update_delay, self.timer_setup)

    def _configure(self, qtile, bar):
        base._TextBox._configure(self, qtile, bar)
        self.setup_images()

    def _get_icon_key(self):
        """Map current battery state to an icon key string."""
        key = 'battery'
        info = self._get_info()
        if info is False or not info.get('full'):
            key += '-missing'
        else:
            percent = info['now'] / info['full']
            if   percent < .1: key += '-empty'
            elif percent < .2: key += '-10'
            elif percent < .3: key += '-20'
            elif percent < .4: key += '-30'
            elif percent < .5: key += '-40'
            elif percent < .6: key += '-50'
            elif percent < .5: key += '-60'   # NOTE: unreachable — kept for compat
            elif percent < .8: key += '-70'
            elif percent < .9: key += '-80'
            elif percent < 1:  key += '-90'
            else:              key += '-full'

            if   info['stat'] == CHARGING: key += '-charge'
            elif info['stat'] == CHARGED:  key += '-charged'
        return key

    def update(self):
        icon = self._get_icon_key()
        if icon != self.current_icon:
            self.current_icon = icon
            self.draw()

    def draw(self):
        if self.theme_path:
            self.drawer.clear(self.background or self.bar.background)
            self.drawer.ctx.set_source(self.surfaces[self.current_icon])
            self.drawer.ctx.paint()
            self.drawer.draw(offsetx=self.offset, width=self.length)
        else:
            self.text = self.current_icon[8:]
            base._TextBox.draw(self)

    def setup_images(self):
        """Pre-scale all icon PNGs to match the bar height."""
        for key, name in self.icons.items():
            try:
                path = os.path.join(self.theme_path, name)
                img  = cairocffi.ImageSurface.create_from_png(path)
            except cairocffi.Error:
                self.theme_path = None
                self.qtile.log.warning('Battery Icon switching to text mode')
                return
            input_width  = img.get_width()
            input_height = img.get_height()
            sp    = input_height / (self.bar.height - 1)
            width = input_width / sp
            if width > self.length:
                self.length = int(width) + self.actual_padding * 2
            imgpat = cairocffi.SurfacePattern(img)
            scaler = cairocffi.Matrix()
            scaler.scale(sp, sp)
            scaler.scale(self.scale, self.scale)
            factor = (1 - 1 / self.scale) / 2
            scaler.translate(-width * factor, -width * factor)
            scaler.translate(self.actual_padding * -1, self.y_poss)
            imgpat.set_matrix(scaler)
            imgpat.set_filter(cairocffi.FILTER_BEST)
            self.surfaces[key] = imgpat

    i = qtile.screens.index(qtile.current_screen)

** multi-monitor group routing
These two functions implement screen-aware group switching: pressing a group
key sends focus to the correct physical screen rather than always switching the
primary screen.  This is particularly important for a 3-monitor setup where
groups are pinned to specific displays.

*Note:* there is a minor bug in ~go_to_group~ — the ~else~ branch calls
~qtile.groups_map.toscreen()~ instead of ~qtile.groups_map[name].toscreen()~.
This is preserved here to match the original, but is flagged as a potential
fix.
#+NAME: config-group-routing
#+BEGIN_SRC python
def go_to_group(name: str):
    """Switch to group *name*, routing focus to the appropriate screen."""
    def _inner(qtile):
        logger.warning(qtile.groups_map)
        if len(qtile.screens) == 1:
            qtile.groups_map[name].toscreen()
            return
        if name == "egrave":
            qtile.focus_screen(2)
            qtile.groups_map[name].toscreen()
        if name == "ampersand":
            qtile.focus_screen(1)
            qtile.groups_map[name].toscreen()
        else:
            qtile.focus_screen(0)
            qtile.groups_map.toscreen()   # BUG: should be groups_map[name]
    return _inner


def go_to_group_and_move_window(name: str):
    """Move the focused window to group *name* and switch to its screen."""
    def _inner(qtile):
        if len(qtile.screens) == 1:
            qtile.current_window.togroup(name, switch_group=True)
            return
        if name == "egrave":
            qtile.current_window.togroup(name, switch_group=False)
            qtile.focus_screen(2)
            qtile.groups_map[name].toscreen()
        if name == "ampersand":
            qtile.current_window.togroup(name, switch_group=False)
            qtile.focus_screen(1)
            qtile.groups_map[name].toscreen()
        else:
            qtile.current_window.togroup(name, switch_group=False)
            qtile.focus_screen(0)
            qtile.groups_map[name].toscreen()
    return _inner

** window cycling hack
This ~@lazy.group.function~ decorator cycles through hidden/overlapping windows
in the current group by bringing them to the front in turn.  It was added to
make Eclipse (which spawns many sub-windows) usable in a tiling environment.
#+NAME: config-cycle-windows
#+BEGIN_SRC python
@lazy.group.function
def cycle_windows(group, forwards=True):
    """Cycle through overlapping windows, bringing each to the front."""
    current = group.current_window
    if current:
        current.change_layer()
    if forwards:
        group.next_window()
    else:
        group.previous_window()
    current = group.current_window
    if current:
        current.bring_to_front()

keys.extend([

        desc="Move window focus to other window"),

** floating layout and global settings
~floating_layout~ defines which windows are always floated by default.  The list
includes common utility windows (dialogs, file-progress indicators, Arandr) as
well as application-specific entries.

~focus_on_window_activation = "smart"~ means Qtile focuses a newly activated
window only if it is on the current screen; otherwise it raises the group bar
urgency hint.  This prevents focus-stealing when background applications signal
readiness.

~wmname = "LG3D"~ is a well-known workaround that causes Java Swing applications
(including Eclipse) to render correctly under non-reparenting window managers.
